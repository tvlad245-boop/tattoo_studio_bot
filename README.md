# Бот студии тату (v1, каркас)

Python 3.10+ (в ТЗ — 3.11+), **aiogram 3.x**, **SQLite** (aiosqlite), FSM: `MemoryStorage` (черновик заявки и шаги анкеты хранятся в БД, при `/start` предлагается «Продолжить» / «Сброс»).

## Ключевые решения (зафиксировано в коде)

### Слоты и вариант A (§5.3)

Слоты общие для студии: заявка хранит `slot_id` и `master_id`. Режим занятости задаётся настройкой `occupancy_mode` в таблице `settings`:

- **`single_room`** (по умолчанию, §8 «одно место»): подтверждённая/ожидающая заявка на слот блокирует слот для **всех** мастеров. Свободные слоты в календаре считаются через `booking_svc.is_slot_available`.
- **`per_master`**: конфликт только по паре `(master_id, slot_id)`; см. `_render_masters` и `is_master_free_on_slot`.

### Окно бронирования (§5.1)

Функция `booking_window_bounds()` в `tattoo_studio_bot/utils/booking_window.py`: от сегодняшнего дня в TZ студии до **конца следующего календарного месяца** включительно.

### Колбэки клиента и админки (§9)

Клиентские callback_data начинаются с `c|`, админские — с `a|` (админ-роутер отделён).

### Ограничение 64 байта

Формирование в `utils/callbacks.py`; при расширении использовать короткие коды и числовые id из БД.

### Картинки разделов (главное меню, «О нас», «Прайс»)

`file_id` хранятся в `settings` (`section_photo_main`, `section_photo_about`, `section_photo_price`). В **/admin** → «Картинки разделов» выбирается экран, боту присылается фото. Подпись — тот же HTML, что и у текстового экрана; лимит Telegram на подпись к фото — 1024 символа.

### Календарь: крестики и «недоступные» дни

Недоступные дни (включая **дни без свободных слотов** в БД) заранее помечаются **❌**; выбор возможен только если есть хотя бы один слот, не занятый по правилам занятости и не помеченный как `studio_blocked`. Так устраняется ситуация «день кажется активным, а записаться нельзя».

### Дальнейшие шаги (не полный объём ТЗ в этой сборке)

Полная админка (редактор анкеты, массовые слоты, заявки, публикация расписания в группу §3.4) — можно наращивать поверх текущего `handlers/admin.py`.

## Запуск

1. Скопировать `.env.example` → `.env`, указать `BOT_TOKEN`, `ADMIN_IDS`.
2. `pip install -r requirements.txt`
3. Из каталога проекта (родитель пакета): `python -m tattoo_studio_bot`

Для смоук-теста добавьте хотя бы один слот на нужную дату в таблицу `slots` (или через будущую админку).

## Группа мастера и расписание (§3.3–3.4)

Для onboarding: создать группу, добавить бота, выдать право отправки сообщений; сохранить `schedule_chat_id` у мастера и использовать идемпотентное поле `schedule_message_id` при реализации `schedule_publish` (заготовка под `edit_message_text`).
